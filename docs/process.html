Simple path from request to delivery.

Step 1

Open a GitHub issue with the public repo URL, the audience, the next action the README should drive, the main weak spot, and one constraint to respect.

Step 2

The request gets a fit check against the narrow pack scope. Out-of-scope work is declined instead of being stretched into the base offer.

Step 3

If the request fits, payment instructions are sent. No delivery promise starts before payment and complete intake.

Step 4

The buyer receives a Markdown delivery with a rewritten opening, three headline directions, one CTA or install rewrite, a short structure fix list, and one first change to ship.

Step 5

Delivery target is within 24 hours after payment and intake. The work remains text-only and AI-assisted throughout.
