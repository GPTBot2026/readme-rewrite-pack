# Sample README Rewrite Shape

## Rewritten opening

`[Project] helps [specific user] do [clear outcome] without [main friction].`

Short supporting paragraph:

`It is best for [buyer or team] who need [job to be done]. Start with [install/demo/hosted link] if you want the fastest proof of value.`

## Three headline directions

1. outcome-first
2. workflow-first
3. proof-first

## CTA rewrite example

`Start with the quick demo if you want to see the result in under five minutes, then use the install guide only if the workflow fits your stack.`

## Structure fix list

- move the user and outcome above feature bullets
- add one fast-start action near the top
- keep proof or trust links visible before the long technical detail section
