# README Rewrite Pack Payment Handoff

Use this file after a request has passed fit-check and the buyer wants to continue with the paid
USD 15 README rewrite pack.

## 1. Standard Payment Request

Use when the request fits and the buyer wants to move forward.

```md
The request fits the README Rewrite Pack.

The next step is USD 15 paid upfront by PayPal.

After payment is confirmed, delivery target is within 24 hours in Markdown.

To continue, send payment to: `[PAYPAL ADDRESS OR LINK]`

Then reply here with:
- confirmation that payment was sent
- the exact public repo URL
- the linked product URL if there is one
- one concern to weight most heavily

What the paid pack includes:
- one rewritten opening section
- three headline or tagline options
- one CTA, install, or demo section rewrite
- a short README structure fix list
- the first change to ship
```

## 2. Short Reminder Reply

Use if the buyer said they would pay but then went quiet.

```md
The README Rewrite Pack is still available if you want to continue.

Current path is the same: USD 15 paid upfront by PayPal, then Markdown delivery within 24 hours
after payment confirmation and complete intake.

If you want to continue, send payment to `[PAYPAL ADDRESS OR LINK]` and reply here with
confirmation plus the exact public repo URL.
```

## 3. Payment Confirmed Reply

```md
Payment confirmed.

I have the repo URL and I am preparing the README rewrite now.

Delivery target: within 24 hours in Markdown.
```

## 4. Internal Notes

- Keep the issue truthful: one public repo or one linked public product page, one paid pack, no hidden upsell.
- If the buyer needs privacy, deeper docs work, or code changes, decline instead of stretching the base offer.
- Reuse the same language across the Pages site, README, issue replies, and delivery flow.
