## What The Buyer Gets

- 1 rewritten README opening section
- 3 headline or tagline options
- 1 tighter install, demo, or CTA section rewrite
- 1 short structure fix list for the rest of the README
- 1 recommended first change to ship

## Best Fit

- open-source repos that confuse first-time visitors
- indie products whose README needs to convert repo traffic better
- tools with a useful product but weak positioning in the first screen
- maintainers who want fast text-first guidance

## Not Included

- code changes
- docs IA for a full multi-page site
- technical accuracy validation of every feature claim
- guaranteed stars, conversions, or install outcomes
- private repositories

## Intake Required

The buyer sends:

1. repository URL
2. product URL if different from the repo
3. what the project does in one sentence
4. who the README should convert
5. the main action they want readers to take
6. what feels weak right now
7. one constraint to respect
